Changes

  • High Priority Changes:
    • Front-End & Back-End are seperated.
      • Seperating layers makes development and deployment simple.
      • Mostly back-end is shared by multible Front-End applications
      • It descreases load on server.
    • Config transformantion is added
      • It avoids manual configuration process
    • Swagger is added as Api documentation
      • Swagger is market standard out-of-box api documentation
      • It helps api-client generation easy.
    • Info Service is addded to display version and stage.
      • It helps to verify deployment
    • Health Service is addded to validate connection with third-party services
      • Health Services is a docker requirement. In case of a service failure; it spins up new instance
      • It helps diagnosis after deployment
  • Technological Changes:
    • Back-End is modified to support docker
      • Docker makes development and deployment easy
      • Isolated docker environment decreases risk of disaster
      • Docker is cheaper than Virtual Image hosting
    • Dotnet Framework replaced with .NetCore 3.0
      • .NetCore is much more lighter and .Net Framework
      • It's cross-platform. So we can take advantages of Linux Images
  • Logical Changes:
    • DTO transformation is moved to client-side.
      • Micro-services supposed to be generic and provided all required information. Consumer transform it on its own side.
      • DTO transoformation logic is part of client-side. Client transforms it based on its requirement.
    • Api url format is chnaged from MVC to REST.
      • Rest is market standard. Applying same standards helps development easy.
  • Bonus Changes:
    • Front-End is converted into Angular App.
      • Angular templating makes development is easy for big projects
  • Suggested Changes:
    • Using GraphQL instead of REST
      • It provides better response custumization and DTO transformantion.
